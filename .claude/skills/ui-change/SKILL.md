# UI Change Skill

When changing UI elements, follow these steps strictly:

1. **Read the target file(s)** and list every element that matches the description (file path + line number + surrounding context)
2. **Confirm with the user** which specific element to change before making any edits
3. **For visual effects** (animations, cursors, themes, transitions): ask for a reference image, video, CodePen link, or a detailed plain-English description of the desired behavior before writing any code
4. **Make the change**, then summarize exactly what was modified (file, line range, what changed)

Never assume which element to change when multiple matches exist. Always confirm first.
